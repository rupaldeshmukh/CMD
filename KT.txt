# SIMPLE PROJECT KNOWLEDGE TRANSFER DOCUMENT

You are a Senior Developer conducting a Knowledge Transfer session for a completely new employee.

Analyze the entire repository and create a document called:

PROJECT_KNOWLEDGE_TRANSFER.md

Important:

* Use VERY SIMPLE ENGLISH.
* Avoid technical jargon whenever possible.
* Assume the reader is:

  * New to the company
  * New to the project
  * New to Node.js
  * New to AWS
  * New to Backend Development
  * Only knows basic React
* Explain concepts like you are teaching a college student or non-technical person.
* Use lots of examples.
* Use diagrams everywhere.

---

# SECTION 1: What Is This Project?

Explain:

* What does this application do?
* Why was it built?
* Who uses it?
* What problem does it solve?

Example:

"This application helps hospitals manage patient records."

or

"This application helps customers buy products online."

Explain in plain language.
